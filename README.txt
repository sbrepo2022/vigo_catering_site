Инструкция по сборке проекта

Для сборки проекта используется webpack и препроцессор scss для сборки стилей в бандл.
Поэтому сначала необходимо клонировать репозиторий (git clone) на компьютер.
Затем в корне рабочей директории выполнить npm install для загрузки зависимостей (для этого понадобится менеджер пакетов npm).
Все необходимые зависимости находятся в package.json, поэтому доустанавливать зависимости не придется.
Для сборки проекта (опять же в корне рабочей директории) нужно выполнить npm run build для единоразовой сборки или npm run start, чтобы
изменения в файлах отслеживались, тогда проект будет пересобираться после каждого изменения автоматически.

В результате сборки, в директории /app/public_html/dist появятся файлы app.min.js и app.min.css, в которых будут содержаться все стили и скрипты.
В директории /app/public_html/src находятся файлы стилей (scss) и скриптов (js), которые можно изменять, после сборки все их содержимое попадет
в файлы, указанные выше.

На сервер нужно загружать файл index.php и /app/public_html/dist, в них содержатся все файлы, необходимые для работы сайта.

Картинки, всякие мультимедийные файлы, какие то скрипты и т. д., в общем все, что будет подключаться напрямую в index.php или в файлах стилей нужно
помещать сразу в /app/public_html/dist, так как в /app/public_html/src хранятся только стили и скрипты, которые после сборки попадут в app.min.js и app.min.css

Точка входа для файлов стилей: /app/public_html/src/styles/app.scss
Точка входа для файлов скриптов: /app/public_html/src/scripts/app.js
